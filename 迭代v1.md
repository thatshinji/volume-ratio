## 一、产品定位

个人交易辅助工具，核心场景：盯盘时看飞书通知，偶尔在终端查数据。

**不做 Web 仪表盘**，强化飞书 + CLI 两个界面。

### 架构总览

```
┌─────────────────────────────────────────────────┐
│  飞书（主界面）                                    │
│  信号卡片推送 / 交互指令 / 菜单 / 定时简报          │

│  CLI（管理界面）                                   │
│  状态检查 / 历史查询 / 标的管理 / 配置              │
├─────────────────────────────────────────────────┤

│  行情采集 / 量比计算 / 信号检测 / 数据清理           │
└─────────────────────────────────────────────────┘
```

## 二、飞书自建应用升级

### 2.1 从 Webhook 升级为自建应用

当前 webhook 只能单向发纯文本。升级为飞书自建应用，一个 App ID 同时搞定收发。

| 能力 | Webhook（当前） | 自建应用（升级后） |
|:--|:--:|:--:|
| 发送文本消息 | ✅ | ✅ |
| 富文本卡片 | ❌ | ✅ |
| 卡片内按钮 | ❌ | ✅ |
| 接收用户指令 | ❌ | ✅ |
| 自定义菜单 | ❌ | ✅ |

### 2.2 接收方式：WebSocket 长连接

不需要回调服务器，不需要公网 IP。本地跑一个 `feishu_bot.py` 守护进程，通过 WebSocket 长连接接收飞书事件，和现有的 `collect_ws.py` 模式完全一致。

```
collect_ws.py   → 连 Longbridge WebSocket → 收行情 → 存文件
feishu_bot.py   → 连飞书 WebSocket        → 收菜单点击 → 回复消息
```

### 2.3 自定义菜单

用户在飞书对话框点击菜单按钮即可发指令，不需要手动输入文字：

```
飞书机器人菜单
├── 📊 查看状态        → /status
├── 🔍 扫描持仓        → /scan
├── 📈 今日信号        → /signals
├── 📋 量比简报        → /brief
└── ⚙️ 管理            → 子菜单
    ├── ➕ 添加标的     → 弹窗输入 ticker
    ├── ➖ 移除标的     → 展示列表选择
    └── 🔕 静默设置     → 展示列表选择
```

### 2.4 交互指令

| 指令 | 功能 | 回复格式 |
|:--|:--|:--|
| `/status` | 系统健康状态 | WebSocket 状态、最近采集时间、今日信号数、磁盘占用 |
| `/scan` | 当前量比快照 | 分市场卡片，按量比排序 |
| `/signals` | 今日触发信号列表 | 信号卡片列表，含时间、标的、类型 |
| `/brief` | 立即发送量比简报 | 同现有 30 分钟简报 |
| `/add CLF.US-克利夫兰` | 添加监控标的 | 确认卡片，热加载生效 |
| `/remove CLF.US` | 移除监控标的 | 确认卡片 |
| `/mute CLF.US 2h` | 静默指定标的 | 静默倒计时卡片 |
| `/history CLF.US` | 近 7 日量比趋势 | 文字趋势摘要 |

### 2.5 富文本信号卡片

信号推送从纯文本升级为卡片，底部按钮支持直接操作：

```
┌─────────────────────────────────────┐
│ 🔥 【日内】CLF.US 克利夫兰          │
│─────────────────────────────────────│
│ 当前价              量比             │
│ $10.48 (↓1.27%)    2.01 ⬆⬆ 放量    │
│─────────────────────────────────────│
│ [LLM分析] CLF放量2.01且下跌1.27%，  │
│ 可能伴随恐慌抛售...                  │
│─────────────────────────────────────│
│ [静默2h]  [查看历史]  [移除监控]     │
└─────────────────────────────────────┘
```

按钮功能：
- **静默 2h**：调用 `/mute CLF.US 2h`
- **查看历史**：调用 `/history CLF.US`
- **移除监控**：二次确认后调用 `/remove CLF.US`

### 2.6 信号去重策略

采用状态机模型，避免轰炸又不漏信号：

```
规则：状态变化时推送，状态持续时静默，状态升级时再推送

示例时间线：
  13:00 量比 0.8 (正常)     → 不推
  13:01 量比 1.8 (放量)      → 推送 "放量开始"  ← 状态变化
  13:02 量比 2.1 (放量)      → 不推             ← 状态持续
  13:03 量比 2.5 (放量止跌)  → 推送 "放量止跌"  ← 状态升级
  13:04 量比 1.2 (正常)      → 不推             ← 回到正常
  13:05 量比 0.5 (缩量止跌)  → 推送 "缩量止跌"  ← 新信号
```

### 2.7 技术要点

- SDK：`lark-oapi`，使用 `WsClient` 长连接模式
- 发送：SDK 直接调用飞书 API，替代 webhook
- 接收：长连接 WebSocket，本地守护进程
- 配置：`config.yaml` 中 `feishu` 节改为 `app_id` + `app_secret`

```yaml
feishu:
  app_id: "cli_xxx"
  app_secret: "xxx"
  # webhook_url 保留作为备用，升级后可移除
```

## 三、数据存储优化

### 3.1 问题

当前每个快照一个 JSON 文件，一天产生 61,500 个文件，实际占用 ~240MB（4KB 磁盘块 × 文件数）。文件数无限增长，`list_snapshots()` 性能持续退化。

### 3.2 方案：JSONL 按标的按天合并

每个标的每天一个 JSONL 文件，追加写入：

```
当前:  data/snapshots/US/CLF_US_20260429_130641_350273.json  (6万+文件/天)
改为:  data/snapshots/US/CLF_US_20260429.jsonl               (11文件/天)
```

JSONL 文件格式（一行一条快照）：

```jsonl
{"ts":"2026-04-29T09:30:01","price":10.41,"volume":10000,...}
{"ts":"2026-04-29T09:30:03","price":10.42,"volume":10500,...}
{"ts":"2026-04-29T09:30:05","price":10.43,"volume":11200,...}
```

### 3.3 存储预算（500MB）

| 数据类型 | 保留策略 | 存储量 |
|:--|:--|:--|
| 日内原始快照（JSONL） | 20 天 | 20 × 18MB = 360MB |
| volume_ratios（计算结果） | 20 天 | ~20MB |
| signals（信号记录） | 20 天 | ~2MB |
| daily_summary（日级汇总） | 90 天 | ~2MB |
| 预留缓冲 | — | ~116MB |
| **合计** | — | **~500MB** |

### 3.4 改造前后对比

| 指标 | 改造前 | 改造后 |
|:--|:--|:--|
| 每日文件数 | 61,500 | 11 |
| 每日磁盘占用 | ~240MB | ~18MB |
| 20 天文件总数 | 1,230,000 | 220 |
| 20 天磁盘占用 | >4GB | ~360MB |

### 3.5 涉及改动

| 文件 | 改动内容 |
|:--|:--|
| `collect_ws.py` `save_snapshot()` | 从 `open+json.dump` 改为追加写入 JSONL |
| `compute.py` 读取函数 | 从遍历目录改为读 JSONL 文件逐行解析 |
| `compute.py` `init_db()` | 新增 `signals` 表 |
| `scripts/cleanup.py` | 新建，数据清理脚本 |
| `start_all.py` | 新增 cleanup cron |

## 四、数据清理策略

### 4.1 动态市场收盘检测

不使用固定时间，脚本自动判断各市场是否已收盘：

```python
def is_market_closed(market: str) -> bool:
    if market == "CN":
        # A股 15:00 收盘，16:30 后清理
        return now.hour >= 16 and now.minute >= 30
    elif market == "HK":
        # 港股 16:00 收盘，17:00 后清理
        return now.hour >= 17
    elif market == "US":
        # 美股 16:00 ET 收盘，自动处理夏令时/冬令时
        et_now = get_et_now()  # 使用 zoneinfo 或 pytz
        return et_now.hour >= 17
```

### 4.2 清理规则

| 数据 | 清理时机 | 保留天数 |
|:--|:--|:--|
| JSONL 快照 | 对应市场收盘后 | 20 天 |
| volume_ratios | 对应市场收盘后 | 20 天 |
| signals | 对应市场收盘后 | 20 天 |
| daily_summary | 每日凌晨 | 90 天 |

### 4.3 Cron 配置

```cron
# 每小时检查一次，脚本内部判断是否已收盘
0 * * * * python3 scripts/cleanup.py >> logs/cleanup.log 2>&1
```

## 九、配置热加载

### 9.1 问题

当前 `load_config()` 使用 `lru_cache` 缓存，修改 `config.yaml` 后必须重启所有进程才能生效。

### 9.2 方案

改为文件修改时间检查，有变化才重新读取：

```python
_config_cache = None
_config_mtime = 0

def load_config():
    global _config_cache, _config_mtime
    mtime = CONFIG_PATH.stat().st_mtime
    if mtime != _config_mtime:
        _config_cache = yaml.safe_load(open(CONFIG_PATH))
        _config_mtime = mtime
    return _config_cache
```

改配置后自动生效，无需重启。`--add`、`--remove`、`--mute`、LLM 切换等操作即时生效。
