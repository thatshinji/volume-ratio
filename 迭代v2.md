## 二、飞书交互升级

### 2.1 /watchlist — 交互式关注列表

发送 `/watchlist` 返回按市场分组的关注列表卡片，每个标的带删除按钮。

```
📋 关注列表
🇺🇸 美股
┌──────────────┬──────┐
│ 标的          │ 操作  │
├──────────────┼──────┤
│ CLF.US-克利夫兰│ [❌ CLF.US] │
│ NVDL.US-英伟达2x│ [❌ NVDL.US] │
└──────────────┴──────┘
```

- 点击删除按钮 → 从 config.yaml + 长桥自选股同步移除
- 卡片实时刷新，删除后立即更新列表

### 2.2 /allstock — 二级导航

发送 `/allstock` 查看长桥所有自选股分组，支持二级导航：

**一级：分组列表**
```
📈 全部股票
📁 纳指科技 — 15 只
📁 中概股 — 8 只
[📁 纳指科技] [📁 中概股]
```

**二级：组内股票**
```
📁 纳指科技（15 只）
AAPL.US-Apple
MSFT.US-Microsoft
...
[➕ AAPL.US] [➕ MSFT.US] ...  (每行 5 个)
[⬅ 返回列表]
```

- 点击添加按钮 → 同步添加到长桥"量比监控"分组 + config.yaml
- 点击返回 → 回到分组列表

### 2.3 卡片回调机制

飞书卡片按钮点击通过 `P2CardActionTrigger` 回调处理，关键技术点：

- **WsClient monkey-patch**：原版 `_handle_data_frame` 静默丢弃 CARD 消息，需 patch 路由到 `event_handler`
- **P2CardActionTriggerResponse**：支持 toast 提示 + 原地刷新卡片
- **飞书应用配置**：需在开发者后台开启"卡片动作回调"

| `/stop` | 关停量比系统 | 关停确认卡片 |

### 2.5 /sync — 长桥持仓同步

## 三、稳定性修复

### 3.1 飞书机器人 daemon 模式

**问题**：双 fork + asyncio kqueue selector 不兼容，daemon 模式崩溃（`OSError: Bad file descriptor`）

**方案**：改用 `os.execv()` 替换进程，避免 kqueue 文件描述符继承问题

```python
# 旧方案：双 fork（不兼容 asyncio）
pid = os.fork()
os.setsid()
pid = os.fork()

# 新方案：execv 替换进程
os.execv(sys.executable, [sys.executable, str(Path(__file__).resolve())])
```

### 3.2 WebSocket 指数退避重试

```python
max_retries = 5
retry_delays = [30, 60, 120, 300, 600]  # 秒
```

连接失败时按退避策略重试，超过最大次数后停止，由 launcher 在下次检查时重启。

### 3.3 /brief 价格 fallback

**问题**：WebSocket 无推送的标的（如 CRML.US）没有快照数据，价格显示 $0

**方案**：`compute_all()` 检测价格为 0 的标的，批量从长桥 API 获取最新价

```python
zero_tickers = [r["ticker"] for r in results if r.get("price", 0) == 0]
if zero_tickers:
    api_prices = _fetch_price_from_api(zero_tickers)
```

### 3.4 parse_ticker 修复

**问题**：`rsplit('-', 1)` 无法正确解析含连字符的名称（如 `1810.HK-XIAOMI-W`）

**方案**：改用正则匹配 ticker.market 格式

```python
# 旧：rsplit('-', 1) → ('1810.HK-XIAOMI', 'W') ❌
# 新：regex r'^([A-Za-z0-9]+\.[A-Za-z]+)-(.+)' → ('1810.HK', 'XIAOMI-W') ✅
```

### 3.5 .pyc 缓存陷阱

git 不保留文件时间戳，导致旧 `.pyc` 缓存比新 `.py` 文件更"新"，Python 加载旧代码。修复方法：代码更新后清除 `__pycache__/`。

## 四、功能增强

### 4.1 /status 显示 LLM 调用次数

新增 `llm_calls` 表记录每次 API 调用，`/status` 显示今日调用次数。

```sql
CREATE TABLE llm_calls (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT NOT NULL,
    model TEXT,
    success INTEGER DEFAULT 1
);
```

### 4.2 LLM max_tokens 调优

从 200 → 500 → 800，避免分析结果被截断。

### 4.3 市场交易时间判断

信号扫描和简报推送增加交易时间判断，非交易时段跳过：

| 市场 | 开盘 | 收盘 | 判断逻辑 |
|:--|:--|:--|:--|
| A股 | 09:30 | 15:00 | 北京时间 |
| 港股 | 09:30 | 16:00 | 北京时间 |
| 美股 | 09:30 ET | 16:00 ET | 自动处理夏令时 |
